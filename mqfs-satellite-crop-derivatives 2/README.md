# Satellite-Indexed Parametric Crop-Insurance Derivatives over Sicily

**MQFS Working Paper · open-source companion code**
*Pricing crop drought insurance from Sentinel-2 vegetation indices — "alpha from space."*

An end-to-end, reproducible pipeline that turns satellite imagery into a priced
parametric insurance contract. It ingests Sentinel-2 NDVI over Sicilian cropland,
isolates the seasonal "yield-shock" signal, models the index as a mean-reverting
process around a fitted climatology, and prices a drought put with a Monte Carlo
engine implemented in **four languages** (NumPy, C, C++, MATLAB) that agree to
within Monte Carlo standard error.

> Independent student research by **Nicolò Angileri**, Mediterranean Quantitative
> Finance Society (MQFS). MIT licensed. Not investment, insurance, or financial advice.

## What it does

The system is three conceptual modules feeding one pricer:

```
            THE ORACLE                  THE QUANT LAYER              THE PRICING ENGINE
   ┌──────────────────────────┐   ┌──────────────────────────┐   ┌──────────────────────────┐
   │ Sentinel-2 L2A (GEE or    │   │ Whittaker smoothing       │   │ OU calibration (closed    │
   │ local Copernicus tiles)   │   │ Harmonic climatology      │   │ form on raw residual)     │
   │ SCL cloud mask            │──▶│ De-seasonalised anomaly   │──▶│ Monte Carlo (NumPy/C/C++) │
   │ cropland mask (WorldCover)│   │ VCI + drought episodes    │   │ SD principle + Wang       │
   │ dekadal median composites │   │ seasonal settlement index │   │ VaR / CVaR, burn cost     │
   └──────────────────────────┘   └──────────────────────────┘   └──────────────────────────┘
                                          synthetic fallback for fully-offline runs
```

The contract: a **put on the seasonal mean NDVI** over the durum-wheat
grain-filling window (day-of-year 75–151). It pays the grower
`min(limit, tick · max(0, K − I))` when the satellite index `I` falls below a
strike `K` placed at the 20th historical percentile.

## Why not risk-neutral pricing

NDVI is **not a traded asset**, so the market is incomplete — there is no
replicating portfolio and no unique risk-neutral measure. Pricing by naive
risk-neutral expectation would be a category error. Instead we use two transparent
incomplete-market principles, both of which reduce to the discounted burn cost in
the no-load limit:

- **Standard-deviation principle:** `Π = e^(−rT)(E[Y] + θ·SD[Y])`
- **Wang (2000) transform:** distorts the loss survival function by the market
  price of risk `λ`.

## Repository layout

```
config/                 master config.yaml (region, contract, model, market)
src/oracle/             Sentinel-2 ingestion (GEE + local) + spectral indices + synthetic generator
src/quant_layer/        smoothing, climatology, anomaly/VCI, drought episodes, volatility matrix
src/pricing_engine/     OU calibration, payoff, Monte Carlo, pricer (SD + Wang + VaR/CVaR)
src/bindings/           ctypes wrappers exposing the C and C++ engines as drop-in pricers
c_core/                 zero-dependency C "audit kernel" (xoshiro256**, Box–Muller)
cpp/                    header-only OpenMP C++17 engine + C ABI shim + optional pybind11
matlab/                 calibration, pricing, Wang transform, burn-cost backtest, figures
tests/                  28-test pytest suite (indices, calibration, pricing, native cross-val)
scripts/run_pipeline.py end-to-end driver -> figures/ + paper/generated/
paper/                  LaTeX working paper (+ generated tables) and compiled PDF
```

## Quickstart

```bash
# 1. Python deps (core only; runs fully offline on synthetic data)
pip install -r requirements.txt

# 2. Build the native C and C++ engines
make native

# 3. Run the test suite (28 tests, incl. NumPy-vs-C-vs-C++ agreement)
make test

# 4. Reproduce all figures and the generated pricing tables
make figures        # == python scripts/run_pipeline.py

# 5. Compile the working paper (needs a TeX distribution)
make paper
```

To ingest **real** Sentinel-2 imagery instead of the synthetic generator, install
the geospatial extras (`conda env create -f environment.yml`) and supply Google
Earth Engine or Copernicus credentials; the `oracle` module then replaces the
synthetic panel with no change downstream.

## Results snapshot

Reference durum-wheat drought put, 200k antithetic paths, strike at the 20th
percentile (`K = 0.452`), `θ = 0.25`, `λ = 0.15`:

| Quantity | Value (EUR) |
|---|---:|
| Fair (burn) value | 2,019 |
| SD-principle premium | 3,303 |
| Wang-transform premium | 2,597 |
| Trigger probability | 22.6% |
| Writer CVaR 95% | 20,524 |

The NumPy, C and C++ engines agree on the fair value to within four Monte Carlo
standard errors — the project's central correctness control. (Numbers are
regenerated by `scripts/run_pipeline.py`; see the working paper for the figures.)

## Multi-language roles

| Language | Role |
|---|---|
| **Python** | orchestration, geospatial ingestion, feature engineering, reference NumPy engine |
| **C** | portable zero-dependency audit kernel, callable via `ctypes` / MATLAB `loadlibrary` |
| **C++** | production OpenMP-parallel engine, identical C ABI + optional pybind11 module |
| **MATLAB** | calibration, pricing, Wang transform, burn-cost backtest, research figures |

## Caveats

- The shipped run uses a **synthetic but realistic** NDVI generator so the
  pipeline, tests and paper are fully reproducible offline. Live data plugs in
  through the `oracle` layer.
- **Basis risk** is intrinsic to index insurance: the payout tracks the satellite
  index, not an individual farm's realised loss.
- This is research code, not a production trading or underwriting system, and not
  financial advice.
